## Project Overview

**Name:** DFPS Site Visit (`dfps-site-visit`)  
**Version:** 0.1.1  
**Stack:** React Native 0.82.1 · React 19.1.1 · TypeScript-first source (`src/`)  
**Purpose:** Mobile app for field inspectors to record on-site property violations, verify GPS presence, and sync reports to a backend when online.

**Key capabilities today:**

- Inspector login via API
- GPS-gated site visit initiation (100 m geofence)
- Residential / commercial violation form
- Offline queue — visits stored in AsyncStorage, auto-uploaded when network returns
- Summary screen after visit completion

**1.2 Auth token persistence** ✅ Done

Auth session persistence is implemented with Keychain (`saveSession`/`loadSession`/`clearSession`), and app boot restores session in `src/App.tsx`.

```
npm install react-native-keychain
```

In `api.ts` after login:

```js
import * as Keychain from 'react-native-keychain';
await Keychain.setGenericPassword('auth', token);
```

In `App.tsx` on mount, attempt silent login:

```js
const creds = await Keychain.getGenericPassword();
if (creds) setUser({ token: creds.password });
```

⚠️ Follow-up still pending: remove `'demo-token'` fallback from `pushSiteVisit` in `src/services/api.ts`.

**1.3 Sync reliability** ✅ Done

Implemented in current code:
- Per-visit retry metadata in storage (`retryCount`, `nextRetryAt`, `paused`, `lastError`)
- Exponential backoff + max retry cap
- Pause on permanent errors
- Manual "Retry failed now" action in pending uploads UI
- Existing sync result toasts/alerts retained

```js
// storage.ts — add retryCount field
const visit = { ...data, retryCount: 0, localId: uuid() };

// syncService.ts
if (visit.retryCount >= 5) {
  markVisitFailed(visit.localId); // new storage method
  continue;
}
await markVisitRetried(visit.localId); // increment count
```

✅ Toast/alert feedback is present for sync and key actions.

### Phase 3 — Missing Features

**3.1 Photo capture** ✅ Done

Implemented in `ViolationFormScreen.tsx`:
- Camera launch via `react-native-image-picker`
- Preview UI + retake flow

```js
import { launchCamera } from 'react-native-image-picker';

const takePhoto = async () => {
  const result = await launchCamera({
    mediaType: 'photo',
    includeBase64: true,
  });
  if (!result.didCancel && result.assets?.[0]) {
    setPhotoBase64(result.assets[0].base64);
  }
};
```

**3.2 Violation type caching** ✅ Done

Implemented with TanStack Query caching (`queryKey: ['violationTypes', scope]`, `staleTime`, `gcTime`) in `ViolationFormScreen.tsx`.

```js
// In ViolationFormScreen.tsx fetchViolationTypes()
const CACHE_KEY = `violation_types_${scope}`;
const cached = await AsyncStorage.getItem(CACHE_KEY);
if (cached) return JSON.parse(cached);

const fresh = await api.getViolationTypes(scope);
await AsyncStorage.setItem(CACHE_KEY, JSON.stringify(fresh));
return fresh;
```

Optional future enhancement: persist query cache to disk for cold-start offline support.
