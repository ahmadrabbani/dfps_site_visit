# DFPS Site Visit — Project Review & Improvement Guide

Generated by Claude Code on 2026-05-06. Covers current state, known issues, and prioritized improvement areas.

---

## Project Overview

**Name:** DFPS Site Visit (`dfps-site-visit`)  
**Version:** 0.1.1  
**Stack:** React Native 0.82.1 · React 19.1.1 · JavaScript (TypeScript configured but not used)  
**Purpose:** Mobile app for field inspectors to record on-site property violations, verify GPS presence, and sync reports to a backend when online.

**Key capabilities today:**
- Inspector login via API
- GPS-gated site visit initiation (100 m geofence)
- Residential / commercial violation form
- Offline queue — visits stored in AsyncStorage, auto-uploaded when network returns
- Summary screen after visit completion

## Improvement Roadmap

### Phase 1 — Stability & Security (do first)

**1.1 Environment configuration**

Move all environment-specific values out of source code.

```
npm install react-native-config
```

Create `.env`, `.env.staging`, `.env.production`:
```
API_BASE_URL=http://103.8.115.199:91/dfps-site/public/index.php
SITE_LAT=31.5204
SITE_LNG=74.3587
GEOFENCE_RADIUS_M=100
```

Replace hard-coded values in `api.js` and `App.js` with `Config.API_BASE_URL`, etc.

---

**1.2 Auth token persistence**

Store the auth token securely so inspectors stay logged in between sessions.

```
npm install react-native-keychain
```

In `api.js` after login:
```js
import * as Keychain from 'react-native-keychain';
await Keychain.setGenericPassword('auth', token);
```

In `App.js` on mount, attempt silent login:
```js
const creds = await Keychain.getGenericPassword();
if (creds) setUser({ token: creds.password });
```

Remove the `'demo-token'` fallback entirely.

---

**1.3 Sync reliability**

Add a retry counter per pending visit and exponential backoff.

```js
// storage.js — add retryCount field
const visit = { ...data, retryCount: 0, localId: uuid() };

// syncService.js
if (visit.retryCount >= 5) {
  markVisitFailed(visit.localId); // new storage method
  continue;
}
await markVisitRetried(visit.localId); // increment count
```

Emit a toast/alert after sync so the inspector knows their data was received.

---

### Phase 2 — Navigation & State

**2.1 React Navigation**

Replace the manual string-based screen switcher with React Navigation. This adds back-gesture support, deep linking, and prevents the "blank screen on typo" bug.

The app uses **`@react-navigation/native-stack`** (native transitions, hardware back on Android). Root wraps `src/App` in **`NavigationContainer`** in `App.tsx`.

```
npm install @react-navigation/native @react-navigation/native-stack react-native-screens react-native-safe-area-context
# iOS only:
npx pod-install
```

Define a typed stack:
```ts
type RootStack = {
  Login: undefined;
  Dashboard: { user: User };
  SiteVisit: { siteScope: Scope };
  ViolationForm: { violations: Violation[] };
  Summary: { visit: CompletedVisit };
};
```

Move screen-level state (violations, siteScope) into route params or a lightweight store rather than lifting everything to App.

---

**2.2 State management**

As the app grows, prop-drilling from `App.js` will become painful. Add Zustand (lightweight, no boilerplate):

```
npm install zustand
```

Create `src/store/visitStore.js`:
```js
import { create } from 'zustand';
export const useVisitStore = create(set => ({
  user: null,
  violations: [],
  addViolation: v => set(s => ({ violations: [...s.violations, v] })),
  reset: () => set({ violations: [] }),
}));
```

### Phase 3 — Missing Features

**3.1 Photo capture**

`react-native-image-picker` is already installed. Wire it up in `ViolationFormScreen.js`:

```js
import { launchCamera } from 'react-native-image-picker';

const takePhoto = async () => {
  const result = await launchCamera({ mediaType: 'photo', includeBase64: true });
  if (!result.didCancel && result.assets?.[0]) {
    setPhotoBase64(result.assets[0].base64);
  }
};
```

Add the captured image preview in the form UI and include `photoBase64` in the violation payload sent to the API.

Required permissions to add in `AndroidManifest.xml`:
```xml
<uses-permission android:name="android.permission.CAMERA" />
```

---

**3.2 Violation type caching**

Cache the violation type list in AsyncStorage so the form works fully offline and loads instantly on repeat opens.

```js
// In ViolationFormScreen.js fetchViolationTypes()
const CACHE_KEY = `violation_types_${scope}`;
const cached = await AsyncStorage.getItem(CACHE_KEY);
if (cached) return JSON.parse(cached);

const fresh = await api.getViolationTypes(scope);
await AsyncStorage.setItem(CACHE_KEY, JSON.stringify(fresh));
return fresh;
```

Invalidate the cache on a schedule (e.g., once per day) or after a successful sync.

---

**3.3 Remove unused dependency**

```
npm uninstall react-native-fs
```

If file-system access is needed later (e.g., exporting PDF reports), re-add it then.

### Phase 4 — TypeScript Migration

Convert source files from `.js` to `.ts`/`.tsx` incrementally. Recommended order:

1. `src/theme/colors.js` → `colors.ts` (trivial, no imports)
2. `src/services/storage.js` → `storage.ts` (define `PendingVisit` type)
3. `src/services/api.js` → `api.ts` (define `User`, `ViolationCategory` types)
4. `src/services/syncService.js` → `syncService.ts`
5. Screens — last, after types are established

Add strict mode gradually:
```json
// tsconfig.json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true
  }
}
```

### Phase 6 — Android / Build Hardening

**6.1 Restore Gradle memory for CI**

The current `Xmx1024m` is a local workaround for a low-RAM Windows dev machine. Set it per-environment instead:

```properties
# android/gradle.properties  (committed)
org.gradle.jvmargs=-Xmx2048m -XX:MaxMetaspaceSize=512m

# local override only on dev machine — add to .gitignore
# android/local.properties or GRADLE_OPTS env var in shell profile
```

**6.2 Enable edge-to-edge for Android 15**

```properties
# android/gradle.properties
edgeToEdgeEnabled=true
```

Then handle insets in your root screen with `react-native-safe-area-context` (already a peer dep of React Navigation).

              ├── LoginScreen
